# WARP.md

Agent spec for `~/phd/corpusEmbed`. This is the canonical contract; `README.md` is
the human quick start.

## What this project is
A clean rebuild of the corpus embedding + topical bucketing that previously lived in
`~/phd/literatureSearch`. It embeds every paper once with SPECTER2 and scores it
against the six bucket centroids defined in `buckets.yaml`.

## Hard boundaries
- `~/phd/literatureReview/data/review.db` is read **read-only** (`mode=ro`) and is the
  sole source of paper metadata. Never write to it.
- Do NOT read from, import, or depend on `~/phd/literatureSearch`. `buckets.yaml`
  supersedes any bucket definitions that once lived there.
- Plain ASCII throughout: titles are transliterated; abstracts are already ASCII.

## The one invariant
There is exactly **one** text preparation for the whole corpus. Every paper and every
bucket centroid is built through the same `make_text` + `Specter2Embedder.embed` path:
`text = title + [SEP] + body`, where `body` is `abstract_verbatim` for papers and the
`description` for buckets (with the first three keywords as the synthetic title). No
preferred/fallback branching — a fallback is a latent second preparation, which is the
exact class of bug this project exists to kill.

## Model
- `allenai/specter2_base` + the `proximity` adapter (loaded via the `adapters`
  library). `set_active=True` on load is not enough; `set_active_adapters("proximity")`
  is what activates it for the forward pass.
- CLS pooling: `last_hidden_state[:, 0, :]`, 768-dim, stored as float32.
- Every row is tagged `embedding_model = 'specter2'`.

## Scoring
Cosine of each paper vector against all six centroids; assign argmax. If the top score
is below `buckets.yaml: floor` (0.5), assign `unclassified`. All six scores are stored
and exported — read the full score vector, not just the winning label. SPECTER2 scores
compress into roughly 0.80–0.96, so the floor is an inert safety net, not a working
discriminator.

## The four asserts (the real fix)
Enforced at write time in `build.py` and re-checked by `embed/validate.py`:
1. `embedding_model == 'specter2'` on every row.
2. Every stored vector is exactly 768-dim.
3. The embedded text included the abstract whenever the paper has one (guards
   title-only regressions).
4. One PDF == one row == one export row, keyed on DOI (not path), with a
   `nodoi-<sha256(filename)>` fallback for the 7 DOI-less rows.

## Buckets are extensible
`buckets.yaml` drives everything. Adding a bucket is: add its block, then
`python -m embed.build` — one extra centroid and a re-score. Paper embeddings do not
change, so no re-embedding is required.

## Commands
```
python -m embed.build [--device cpu|cuda] [--review-db ...] [--buckets ...]
                      [--out-db ...] [--export-dir ...]
python -m embed.validate [--db ...] [--export-dir ...]
```
